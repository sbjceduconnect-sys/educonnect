\section{TESTING AND VALIDATION}
\hline

This chapter describes the testing strategies employed to validate the functionality, security, and performance of the EduConnect system. A combination of manual testing, functional testing, and security testing was conducted across all modules.

\subsection{Testing Methodology}

The testing process followed a structured approach:

\begin{enumerate}[label=\arabic*., leftmargin=2em]
    \item \textbf{Unit Testing:} Individual functions and model methods (e.g., grade computation, token generation) were tested in isolation.
    \item \textbf{Integration Testing:} API endpoints were tested end-to-end using tools such as Postman and the Django test client, verifying correct request handling, authentication, authorization, and response formatting.
    \item \textbf{Functional Testing:} Each module was tested against its functional requirements, verifying that all features work as specified.
    \item \textbf{Security Testing:} Authentication, authorization, and data access controls were tested by attempting unauthorized access to protected endpoints.
    \item \textbf{User Interface Testing:} The frontend was tested across multiple browsers (Chrome, Firefox, Edge) and screen sizes (desktop, tablet, mobile) to ensure responsive behavior.
    \item \textbf{Cross-Platform Testing:} The Android APK was tested on multiple devices to verify native functionality.
\end{enumerate}
